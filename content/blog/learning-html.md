HTML 是网页的骨架。很多开发者觉得 HTML 很简单，无非就是一堆标签的堆砌。但事实上，用好 HTML 语义化标签，比你想象的要重要得多。

## 什么是语义化

语义化指的是用**有意义**的标签来描述内容的结构和含义，而不是单纯地用 `<div>` 和 `<span>` 来布局。

HTML5 引入了一系列新的语义化标签，包括：

- `<header>` — 页面或区块的头部
- `<nav>` — 导航链接区域
- `<main>` — 页面的主要内容
- `<article>` — 独立的内容单元
- `<section>` — 主题分组
- `<aside>` — 侧边栏或补充内容
- `<footer>` — 页面或区块的底部

## 为什么要语义化

### 1. 无障碍访问（Accessibility）

屏幕阅读器和辅助技术依赖于语义化标签来理解页面结构。使用正确的标签可以让视障用户更好地浏览你的网站。

### 2. SEO 优化

搜索引擎的爬虫通过分析 HTML 结构来理解页面内容。语义清晰的页面更容易被正确索引。

### 3. 代码可维护性

全是一堆 `<div>` 的页面就像没有路标的城市。语义化标签本身就是文档，让代码更容易阅读和维护。

> 好的 HTML 结构应该是在不看任何 CSS 的情况下，文档的层次结构依然是清晰可读的。

## 常见的语义化错误

在实际项目中，我经常看到以下问题：

- **滥用 `<div>`** — 几乎所有内容都用 div 包裹，没有任何语义信息
- **标题层级跳跃** — 从 `<h1>` 直接跳到 `<h4>`，破坏了文档大纲
- **用 `<br>` 换行** — 应该使用 `<p>` 或 CSS 来控制间距
- **按钮用 `<div>` 实现** — 应该使用 `<button>` 或 `<a>`

## 实践建议

在写 HTML 时，可以遵循一个简单的原则：**先语义、后样式**。先用最合适的标签构建内容结构，然后再用 CSS 来实现视觉呈现。

## 总结

HTML 语义化不是一种"最佳实践"，而是现代网页开发的**基本要求**。它关系到无障碍、SEO、代码质量等多个方面。
